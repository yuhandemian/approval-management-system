## 1. 사업 개요

### 1.1 배경 및 목적

○○시청은 현재 종이 기반 결재 및 전화·방문 중심의 민원 처리 방식을 운영 중이다.
이로 인해 결재 지연, 민원 처리 현황 파악 어려움, 문서 분실 등의 문제가 지속적으로 발생하고 있다.

본 사업은 전자결재 및 민원 처리 업무를 전산화하여 행정 효율을 높이고,
직원과 민원인 모두에게 투명한 업무 처리 환경을 제공하는 것을 목적으로 한다.

### 1.2 사업 범위

- 전자결재 시스템 구축 (기안 → 결재선 지정 → 승인/반려 → 완료)
- 민원 처리 시스템 구축 (신청 → 접수 → 배정 → 처리 → 완료)
- 사용자 및 부서 관리 기능
- 공지사항·자료실·Q&A 게시판
- 권한별 메뉴 접근 제어

### 1.3 적용 기술 표준

| 구분 | 기술 |
|------|------|
| 프레임워크 | 전자정부프레임워크 (eGovFrame) 4.3 |
| 언어 | Java 17 (개발환경) / Java 8 타깃 (운영환경) |
| View | JSP + JSTL |
| Persistence | MyBatis |
| DB | MySQL 8.x |
| WAS | Apache Tomcat 9.x |
| 빌드 | Maven |

## 2. 사용자 역할 및 권한

| 역할 | 코드 | 설명 | 접근 가능 메뉴 |
|------|------|------|----------------|
| 시스템 관리자 | ROLE_ADMIN | 전체 시스템 관리 | 전체 |
| 결재자 | ROLE_APPROVER | 결재 승인/반려 권한 보유 | 결재함, 민원처리, 게시판 |
| 담당자 | ROLE_HANDLER | 민원 처리 담당 | 민원처리, 게시판 |
| 일반 사용자 | ROLE_USER | 기안 작성, 민원 신청 | 기안작성, 민원신청, 게시판 |

> 권한은 중복 부여 가능 (예: 결재자 + 담당자)

## 3. 기능 요구사항

### 3.1 공통 / 로그인

| ID | 요구사항 | 우선순위 |

```
[기안 작성] → [결재 대기] → [승인] → [최종 승인 완료]
                         ↘ [반려] → [반려 처리 완료]
```

| 상태 코드 | 상태명 | 설명 |
|-----------|--------|------|
| DRAFT | 임시저장 | 기안 작성 중 |
| PENDING | 결재 대기 | 결재선에 상신됨 |
| IN_PROGRESS | 결재 중 | 일부 결재자 승인 완료 |
| APPROVED | 최종 승인 | 모든 결재자 승인 |
| REJECTED | 반려 | 결재자 반려 처리 |
| CANCELED | 취소 | 기안자 취소 |

## 6. 제약 사항

- 전자정부프레임워크 4.3 표준 준수 필수

## 7. 용어 정의

| 용어 | 설명 |
|------|------|
| 기안 | 결재를 요청하기 위해 작성하는 문서 |
| 결재선 | 기안서를 승인할 결재자들의 순서 목록 |
| 전결 | 상위 결재자가 하위 결재자에게 결재 권한을 위임 |
| 민원 | 시민이 행정기관에 처리를 요청하는 사항 |
| 담당자 배정 | 접수된 민원을 특정 직원에게 처리 책임 부여 |
