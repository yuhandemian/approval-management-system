## 1. URL 설계 규칙

```
/{모듈}/{동사}{화면명}.do

예시)
  /approval/apvlList.do       → 결재 목록
  /approval/apvlRegist.do     → 기안서 작성
  /approval/apvlDetail.do     → 기안서 상세
  /approval/apvlUpdate.do     → 기안서 수정 처리 (POST)
```

| 동사 접미사 | 의미 |
|-------------|------|
| List | 목록 조회 |
| Detail | 상세 조회 |
| Regist | 등록 화면 / 등록 처리 |
| Update | 수정 처리 |
| Delete | 삭제 처리 |
| Action | 특정 상태 변경 처리 |

## 5. 화면 전환 흐름

### 5.1 전자결재 흐름

```
[기안서 작성] → (임시저장) → [내 기안 목록] → [기안서 상세]
            → (상신)     → [결재함 목록(결재자)] → [결재 처리]
                                              → 승인 → 다음 결재자 / 최종 승인 완료
                                              → 반려 → 기안서 REJECTED
```

### 5.2 민원 처리 흐름

```
[민원 신청] → [내 민원 목록]
           ↓
[민원 현황(관리자)] → 접수 → 담당자 배정 → [민원 현황(담당자)] → 처리완료
                                                             → 반려
```

### 5.3 게시판 공통 흐름

```
[목록] → [상세] → [수정/삭제] (본인 또는 관리자)
[목록] → [작성] → [등록] → [목록]
```

## 6. 공통 화면 컴포넌트

| 컴포넌트 | 적용 화면 | 설명 |
|----------|-----------|------|
| 페이징 | 목록 전체 | eGovFrame PaginationRenderer 활용 |
| 키워드 검색 | 목록 전체 | 제목·내용 LIKE 검색, searchCondition / searchKeyword |
| 파일 업로드 | 기안서·민원·게시글 작성 | 멀티파트, 10MB 제한 |
| 파일 다운로드 | 상세 전체 | ATTACH_ID로 단건 다운로드 |
| 확인 모달 | 삭제·반려·취소 처리 | JavaScript confirm 또는 공통 모달 |
| 권한 체크 인터셉터 | 전체 | Spring HandlerInterceptor, 미인증 → 로그인 리다이렉트 |
